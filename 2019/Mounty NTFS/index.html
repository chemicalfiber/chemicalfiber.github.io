免费的MacOS NTFS磁盘挂载工具：Mounty for NTFS

发表于2019-10-19更新于2021-03-10

Well，MacOS原生默认不支持NTFS文件系统的磁盘读写，今天分享一款软件给大家，让你的Mac设备开启NTFS读写功能！

这里是化学纤维！一位喜欢折腾电脑和手机操作系统的机友！

我们知道，NTFS因为没有FAT32文件系统4G单文件最大限制和比exFAT更好的稳定性而拥有较大的普及率。
但是成功吃上了黑苹果，或者是正在使用Mac设备的朋友们却发现一件非常蛋疼的事情：Mac默认不支持NTFS文件系统的写入，只支持读取。我们安装或者购买Mac，大多数是为了办公用途，而办公过程中免不了用到U盘等储存介质进行文件的交换，但是如果你的同事或朋友给你的U盘是NTFS文件系统的话……

Mac设备如何开启NTFS读写？

虽然说MacOS原生是支持开启NTFS的，但是一大堆的命令已经让小白望而却步。有没有一些挂载工具呢？有啊！先交钱！

百度搜索“Mac NTFS”关键词，得出的第一个软件Paragon NTFS for Mac：¥149。呵呵，怎么不再提高100元再弄个1元加购赠品凑满250？

便宜点的，有Tuxera NTFS for Mac：¥99。WDNMD我还不如再Mac里弄个Windows虚拟机，Oracle VM VirtualBox虽然麻烦点，但是免费还能保证数据安全。
破解版？不存在的！
仔细一看，这两家软件的代理都是那臭名昭著的业界毒瘤：思杰马克丁，对，你花钱还不一定买得到正版的那家软件代理商。当你试图在你访问得到的网站上下载所谓的“破解版”的时候，请留意一下浏览器地址栏，当你点击“下载”按钮的时候，地址栏是不是显示了以 http://wm.makeding.com 作为开头的网址？恭喜，你下载到的就是个“试用版”，永远无法破解。
而且大家还可以去看看这俩软件的《售后服务条款》，简直就是花钱找shi吃。

难道我们穷人就没有人权？

有开发者站出来了！
今天为大家推荐一款比Paragon NTFS for Mac和Tuxera NTFS for Mac更好的NTFS磁盘读写挂载工具。
最重要的是！它是免费的！
它的名字是：

Mounty for NTFS

我们来看看它官网上的描述：

Mounty for NTFS comes without any decorations.
No additional driver will be installed. It simply makes use of the build-in mechanisms already provided by Mac OSX Mavericks - no strings attached.

翻译过来就是（由Google翻译，化学纤维润色）：

Mounty for NTFS没有任何花里胡哨的内容。
它不会安装其他驱动程序。它只是利用了Mac OSX Mavericks（10.9）已经提供的内置机制-不附加任何字符串。

舒服了，免费，微小（3.9MB）的安装包，软件界面语言高度本地化（我的是繁体中文，所以Mounty for NTFS的语言界面跟随了系统语言）。

不过可能是因为“小”的原因，安装引导界面也是标准的Finder界面。只有Mounty for NTFS本体，“应用程序”快捷方式和Read Me First.md文档（这是个更新日志文档，内容全英文，就不用看了）。不过我是非常喜欢这种“极简”风格的。

如何使用Mounty for NTFS？

根据安装镜像里的使用指引，它只要把Mounty for NTFS本体拖动到“应用程序”目录（Applications），即可使用。打开Mounty for NTFS之后你可以在屏幕顶端的菜单列表看到一个山脉图标，就长这样。

点击它，就可以对目前已经连接的NTFS磁盘进行读/写挂载操作了。
当然，如果你希望它可以开机自动挂载你的NTFS本地磁盘，你可以勾选上“开机自启”的选项（简体中文的版本我没用过，不过这个选项就在这个位置），并且选择你的NTFS硬盘分区为“自动挂载”。下次开机进入MacOS之后，选择的分区会自动挂载并且在Finder中打开。

注意事项

被挂载的NTFS分区将在Finder窗口左侧的“位置”列表中消失，你可以在“终端”输入以下命令，在桌面建立一个名为Volumes的快捷方式，用于显示所有驱动器。
sudo ln -s /Volumes ~/Desktop/Volumes
输入上述代码后，后记得输入系统密码确认更改。

当你想拔出已经挂载的NTFS驱动器时，请先在Mounty for NTFS中卸载你的驱动器，然后再使用Finder的“推出”功能。

常见问题

Q1：使用一段时间后，曾经可以成功被挂载的分区现在无法挂载了？
A1：根据官网上的解释，是因为上次使用完此分区后，并没有正常“弹出”，请将此分区在Mac或Windows上正常地“弹出”一次即可。

Q2：上述分区是硬盘上的，不是可移动磁盘分区，怎么弹出？
A2：启动到MacOS，打开Finder，在左侧的磁盘列表中使用鼠标右键点击对应分区，选择“推出”，重启MacOS即可。

Q3：按上述步骤做了之后，还是没有用？
A3：可能是你的NTFS磁盘有错误，需要使用chkdsk命令修复。请在Windows操作系统下，以“管理员身份”打开CMD，输入chkdsk 「磁盘盘符」: /f
例如我需要修复我的D盘，则上述命令为chkdsk d: /f
如果出现“是否卸除该卷”的提示，请按下Y，CMD会进行错误检查并自动修复。
如果依旧无法挂载，请重复A2的步骤。

Q4：Windows+Mac双系统，可以挂载Windows所在分区进行读写吗？
A4：可以，但不建议这么做。

Q5：在Mac上挂载NTFS分区读写，这是否会对我的NTFS磁盘造成伤害？
A5：严格来说，无论是在什么操作系统上对NTFS磁盘进行写入操作都会对其有伤害，所以请不要问这种问题。如果出现数据丢失，请尝试A3的操作步骤。
